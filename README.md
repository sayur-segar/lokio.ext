## Requirements

- [Lokio CLI](https://lokio.dev/endocs/getting-started) installed and available in `PATH`
- A workspace with `lokio.yaml` or `lokio/configs.yaml`

Install the CLI:

```bash
# Using bun
bun install -g lokio
 
# Using npm
npm install -g lokio
 
# Verify installation
lokio --version
```

## Features

### Template Explorer

A dedicated panel in the Explorer sidebar shows all your templates and generated files.

- **Templates** — browse available templates, their parameters, and output files
- **Generated Files** — see every file tracked in `lokio.lock`, grouped by template
- Click any file to open it directly in the editor
- Click any template file to open the source template

### Generate from Template

Run `Lokio: Generate from Template` from the Command Palette (`Cmd+Shift+P`) to pick a template from a searchable list and generate code interactively in the terminal.

You can also click the **▶ Generate** button next to any template in the Explorer tree.

### Standards Check

Lokio checks your codebase against template rules and reports violations inline.

- Diagnostics appear in the **Problems** panel (`Cmd+Shift+M`)
- Violations are categorized as **Warning** or **Error** depending on `dangerMode`
- Naming violations are reported per file
- The **status bar** shows a live summary: `✔ 12/12` or `⚠ 2` or `✖ 1`

Check runs automatically:
- On workspace load
- On every file save (configurable)
- When `lokio/configs.yaml` changes

### CodeLens on Generated Files

Files tracked by `lokio.lock` show inline CodeLens at the top:

```
$(package) Generated by Lokio · my-template    $(sync) Sync    $(symbol-field) name=MyComponent, type=page
```

- **Generated by Lokio** — shows template name and generation date
- **Sync** — re-generates this file using 3-way merge (preserves your edits)
- **Parameters** — shows the parameter values used when the file was generated

### Sync Generated Files

| Command | Description |
|---|---|
| `Lokio: Sync Generated Files` | Re-generates all tracked files with 3-way merge |
| `Lokio: Preview Sync (Diff)` | Preview changes before applying (`lokio diff`) |

### Doctor

Run `Lokio: Run Doctor` to diagnose your Lokio installation and workspace configuration (`lokio doctor`).

### AI Standards Check

Run `Lokio: AI Standards Check` to use AI-powered analysis on your codebase (`lokio ai`).

## Commands

| Command | Description |
|---|---|
| `Lokio: Generate from Template` | Pick and run a template interactively |
| `Lokio: Check Repository` | Run standards check and show diagnostics |
| `Lokio: Sync Generated Files` | Sync all generated files (3-way merge) |
| `Lokio: Preview Sync (Diff)` | Preview what sync would change |
| `Lokio: Run Doctor` | Diagnose your Lokio setup |
| `Lokio: AI Standards Check` | AI-powered standards analysis |
| `Lokio: Refresh` | Refresh tree view, CodeLens, and diagnostics |

## Settings

| Setting | Default | Description |
|---|---|---|
| `lokio.executablePath` | `"lokio"` | Path to the lokio binary (if not in PATH) |
| `lokio.checkOnSave` | `true` | Auto-run check when files are saved |
| `lokio.showStatusBar` | `true` | Show Lokio status in the status bar |

To use a custom binary path:

```json
{
  "lokio.executablePath": "/usr/local/bin/lokio"
}
```

## Status Bar

The status bar item shows the current check result at a glance:

| State | Display |
|---|---|
| All passed | `$(shield) Lokio $(check) 12/12` |
| Warnings | `$(shield) Lokio $(warning) 3` (yellow) |
| Errors | `$(shield) Lokio $(error) 1` (red) |
| Running | `$(sync~spin) Lokio` |

Click the status bar item to run a check manually.

---

## Getting Started

1. Install the [Lokio CLI](https://lokio.dev/en/docs/getting-started)
2. Open a project and run `lokio init` in the terminal
3. The Lokio panel appears automatically in the Explorer sidebar
4. Use `Cmd+Shift+P` → `Lokio: Generate from Template` to scaffold code
